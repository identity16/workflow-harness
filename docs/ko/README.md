# Workflow Harness

[English](../../README.md) | [한국어](README.md)

> 일하는 방식을 바꾸지 않고, AI의 힘을 활용하세요.

## Why

AI는 강력합니다. 하지만 업무에 도입하려면 여전히 처음부터 다시 시작하는 느낌입니다 — 새로운 도구, 새로운 프로세스, 새로운 습관. 게다가 대부분의 AI 도구는 팀 전체가 동의해야 한 사람이라도 가치를 얻을 수 있습니다.

지금 당장 *나 혼자*, 아무에게도 허락받지 않고 시작할 수 있다면?

**Workflow Harness**는 기존 도구 위에 올라가는 AI 강화 인터페이스입니다. Linear(또는 Asana, Jira) 계정을 연결하면, 같은 이슈, 같은 보드, 같은 워크플로우를 그대로 사용합니다 — 다만 뒤에서 AI가 조용히 모든 것을 강화합니다. 팀원들은 Linear를 평소처럼 계속 씁니다. 알 필요도 없습니다. 그저 당신이 일을 더 잘하고 있다는 걸 느낄 뿐입니다.

어떻게 하냐고 물으면, 그들도 원하게 됩니다. 이것이 도입이 퍼지는 방식입니다.

## What

Workflow Harness는 기존 작업 관리 도구에 연결하여 AI로 강화하는 오픈소스 **통합 UI**입니다. 두 가지 핵심 원칙을 따릅니다:

### 모든 산출물에는 Trust Level이 있습니다

AI가 생산하는 모든 산출물은 `ai-generated` 또는 `human-verified`로 태깅됩니다. AI가 혼자 한 것과 내가 확인한 것을 항상 구분할 수 있습니다. 이것은 기능이 아니라, 시스템 전체를 관통하는 철학입니다.

### 경계는 내가 통제합니다

**Trust Policy**는 AI가 자율적으로 행동할 수 있는 곳과 내 확인이 필요한 곳을 정의합니다. 하지만 직접 설정하지 않아도 됩니다 — **시스템이 나에게서 배웁니다**. 처음엔 모든 것에 확인을 요청하다가, 패턴을 관찰하고 ("'chore' 이슈를 항상 승인하시네요"), 정책 변경을 제안합니다. 한 번의 클릭으로 확인. 설정 파일도, 라벨 매핑도 필요 없습니다.

```
1주차:  AI가 모든 것을 물어봄
3주차:  "chore는 항상 승인하시네요. 자동 보강할까요?" → "응"
6주차:  "docs도요?" → "응"
        정말 내 관심이 필요한 것만 검토하게 됩니다.
```

## 동작 방식

```
Workflow Harness를 엽니다 (Linear에 연결된 상태)
        │
  이슈를 만들거나 봅니다
  (Linear와 양방향 동기화)
        │
  Context Engine 작동
  ├─ 관련 코드, 문서, 과거 이슈 수집
  └─ Trust Policy 확인
        │
  ┌─────┴─────┐
  │            │
ai-only    human-required
  │            │
자동       인라인 Q&A
보강      (최소한의 질문,
  │        UI 안에서 바로)
  └─────┬──────┘
        │
  신뢰 수준 메타데이터가
  포함된 보강된 이슈
        │
  Linear로 동기화
  (팀원들은 더 풍부한 이슈를 봄)
```

Linear에 있는 팀원들은 더 잘 작성된, 컨텍스트가 풍부한 이슈를 보게 됩니다. 어떻게 된 건지 알 필요도 없습니다.

## 철학

- **Zero Friction** — 도구를 바꾸지 않습니다. 같은 데이터, 같은 워크플로우. 다만 AI가 내장된 더 나은 인터페이스.
- **Observable** — AI의 모든 행동은 보이고, 추적 가능하고, 설명 가능합니다.
- **Human-in-the-Loop** — 내가 경계를 정하고, AI는 그 안에서 움직입니다.
- **Individual-first** — 한 사람이 오늘 시작할 수 있습니다. 팀 도입은 명령이 아니라 결과를 통해 자연스럽게 일어납니다.
